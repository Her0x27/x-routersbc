1. Первым делом необходимо создать структуру папок проекта.
2. В корне проекте создать README.md Общий промпт разработки.
3. В каждой корневой папке создать README.md с промптом разработки части которая содержит та или иная папка

README.md в каждой папке (см. ниже). Должен содержать промпт для реализации структуру, список функций с описанием и т.д.. План реализации разбитых на задачи:

Использование Go 1.23 + Echo Framework 
Web сервер: 
• 5000 / HTTP 
• Без поддержки HTTPS 
• Аккаунт администратора по умолчанию: sbc:hash(sbc) - # хэш-пароль. После первого входа выводить алерт - который рекомендует сменить пароль в целях безопасности ссылкой на страницу изменения пароля.
• Сессии хранить 24ч в routersbc.sqlitedb
• Страницы для ответов кодов: 404, 501 # templates/{404, 501, bad codes ...}.html
• Обязательная централизованная обработка ошибок/исключений.
 
ЗАПРЕЩАЮ: 
— Использовать фиктивные/симулированные данные 
— Использовать конструкции кода которые игнорируют ошибки
 
Все шаблоны должны быть на основе Tabler.io (общая основа для всех шаблонов). 
Шаблоны не должны содержать статические данные, только РЕАЛЬНЫЕ ДАННЫЕ!
